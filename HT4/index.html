EC2 Networking

Hometask 4

Build a custom VPC for EC2 and prove the result through automatic checks instead of screenshots.

Requirements

  1. Create a VPC named epic-network with a non-default CIDR.
  2. Create 2 public and 2 private subnets.
  3. Create and attach route tables correctly.
  4. Private subnets must not have a default route to an Internet Gateway.
  5. Launch an EC2 instance in one of the public subnets using a new security group.
  6. Create or update the EC2 role used for Session Manager and attach a custom read-only EC2 policy.
  7. Use User data to install nginx, create devops, expose index.html and check.json, and refresh the check file every minute.
  8. Use IMDSv2 and AWS CLI inside the instance to validate the network configuration.

What Must Be Visible Remotely

When I open http://<public-ip>/, I must see a custom page with your name or email, the marker string EC2-HT4-OK, the VPC ID, the subnet ID, and the path /check.json.

When I open http://<public-ip>/check.json, I must receive valid JSON that describes the instance network placement, subnet counts, instance profile state, and service checks.

Submission

Submit only the public IP of your EC2 instance.

How It Will Be Checked

  • http://<public-ip>/ returns HTTP 200.
  • The main page contains EC2-HT4-OK.
  • The main page contains your student identifier.
  • http://<public-ip>/check.json returns valid JSON.
  • The instance reports a custom VPC, 4 subnets total, 2 public subnets, and 2 private subnets.
  • Private subnets do not have a default route to an Internet Gateway.
  • The instance profile is attached.
