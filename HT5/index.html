Docker + HTTPS

Hometask 5

Run a custom HTTPS-enabled nginx server inside a Docker container on EC2 and prove the result through automatic checks.

Requirements

  1. Use a basic Ubuntu EC2 image.
  2. Install Docker and all required packages yourself.
  3. Build the final container from a basic Ubuntu image.
  4. Install and configure nginx inside the container yourself.
  5. Configure HTTP to redirect to HTTPS.
  6. Serve a custom page and /check.json over HTTPS.

Important Container Note

The main process inside a container must stay in the foreground. For nginx, use:

nginx -g 'daemon off;'

What Must Be Visible Remotely

http://<public-ip>/ must redirect to HTTPS.

https://<public-ip>/ must return a custom page with your student identifier and the marker EC2-HT5-OK.

https://<public-ip>/check.json must return valid JSON that reports Docker, container, HTTPS, redirect, and content checks.

Submission

Submit only the public IP of your EC2 instance.
